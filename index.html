ALL SERVERS HOSTED AT QUAKE.CSH.RIT.EDU

Since November 2025, RIT's Computer Science House has been hosting 24/7 Quake servers. We run Deathmatch, Team DM, Capture the Flag, and other game modes. Our servers are open to the public and we host LAN parties semi-regularly.

All servers are maintained by Nick Gilbert, a member of CSH. If you have any questions, suggestions, or issues, please don't hesitate to contact him at paradox@csh.rit.edu.

SERVERS

Quake 1 (NetQuake)

quake.csh.rit.edu:26000

Mostly vanilla Quake 1 server using the NetQuake protocol on QSS-M software, running a few game modes:

  • Deathmatch (DM)
  • Team Deathmatch (TDM)
  • Capture the Flag (CTF)
Requires Threewave CTF for Capture the Flag, which can be downloaded from ModDB.

Best for local play at RIT. Frag limit 20 and no time limit. Runs vanilla maps for DM/TDM and base Threewave maps for CTF. Will support spectator mode soon.

Recommended clients:

  • QSS-M, the leading NetQuake port for Windows, Mac, and Linux
  • Ironwail, a modern high-performance port for Windows and Linux
  • Mark V, an older port with a software renderer for Windows, Mac, and Linux

Quake 1 (QuakeWorld)

quake.csh.rit.edu:27500

Work-in-progress Quake 1 server using the QuakeWorld protocol on MVDSV software, running Deathmatch only at the moment.

Much better netcode for long-distance play, but also features different physics compared to NetQuake. Frag limit 20 and no time limit. Runs vanilla maps. Currently doesn't have CTF, an MotD, or increased axe damage from the NetQuake server, but these will be added soon.

Recommended clients:

  • ezQuake, the leading QuakeWorld port for Windows, Mac, and Linux
  • ezQuake is the only client worth using for QuakeWorld
  • If you plan on playing other QuakeWorld servers, nQuake will help you jumpstart your configuration, but it isn't required for Quake @ CSH
